# Documentation Rules

## What To Update

- Update `README.md` when install, runtime, build, or release behavior changes.
- Update `docs/ai/product-knowledge.md` when product behavior changes.
- Update `docs/ai/issues.md` for durable bug investigation context.
- Update `docs/ai/features.md` when expected user behavior changes.

## Style

- Prefer concise Markdown.
- Use exact file paths and commands.
- Separate facts from recommendations.
- Keep temporary notes out of permanent docs unless they explain a durable
  decision.
